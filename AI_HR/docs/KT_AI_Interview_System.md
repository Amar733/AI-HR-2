# AI Interview System - KT Technical Document

## 1. Purpose

This document explains how the AI interview system works end-to-end in this repository, from job setup to candidate interview, AI analysis, and recruiter review.

Scope covered:

- Core backend APIs and controllers
- Frontend interview runtime flow
- Data model and billing implications
- Operational dependencies and known implementation gaps

## 2. High-Level Architecture

- Frontend: Next.js Pages Router (`frontend/`)
- Backend: Express + MongoDB (`backend/`)
- AI runtime:
  - OpenAI Realtime API for live interview conversation
  - OpenAI Chat Completions API for post-interview analysis
- Storage:
  - MongoDB for jobs/sessions/users/transactions
  - Local filesystem for chunked video uploads (`backend/uploads/`)

Core entry points:

- Backend server bootstrap: `backend/server.js`
- Candidate interview UI: `frontend/pages/interview/[interviewId].js`
- Job management APIs: `backend/routes/jobs.js`
- Session runtime APIs: `backend/routes/interviewSessions.js`

## 3. Core Data Entities

### 3.1 Job

File: `backend/models/Job.js`

- Owns interview configuration and question bank.
- Key fields:
  - `interviewLink` (public candidate URL token)
  - `interviewMode` (`mock` | `real`)
  - `requiredSkills`, `questionTypes`, `customQuestions`, `aiQuestions`
  - `duration`, `totalQuestions`
  - `settings.randomizeQuestions`, `settings.antiCheat`
  - Analytics counters: `totalInterviews`, `completedInterviews`, `averageScore`, `passRate`

### 3.2 InterviewSession

File: `backend/models/InterviewSession.js`

- One candidate attempt per session.
- Key fields:
  - `sessionId` (public/internal unique runtime ID)
  - `ephemeralKey` (OpenAI Realtime temporary key)
  - `status` (`started`, `completed`, etc.)
  - `redFlags[]`
  - `fullTranscription`
  - `analysis` (scores, recommendation, strengths/weaknesses)
  - `videoPath`

### 3.3 User + Wallet

File: `backend/models/User.js`

- Recruiter wallet stores time balance:
  - `wallet.minutesBalance`
  - `wallet.totalMinutesUsed`
  - `wallet.totalMinutesPurchased`

### 3.4 Transaction

File: `backend/models/Transaction.js`

- Ledger/audit for minute usage and wallet changes.
- Usage during interview completion is recorded as negative minutes.

## 4. End-to-End Functional Flows

### 4.1 Recruiter creates AI interview job

1. Frontend creates job via `POST /api/jobs`.
2. Backend persists Job, auto-generates `interviewLink` and `accessCode`.
3. Recruiter may call `POST /api/jobs/:id/generate-questions`.
4. AI questions generated by `AIAnalysisService` (`backend/services/aiAnalysisService.js`) and saved to Job.

Routes:

- `backend/routes/jobs.js`
- Controller: `backend/controllers/jobController.js`

### 4.2 Candidate starts interview

Frontend file: `frontend/pages/interview/[interviewId].js`

1. Candidate opens `/interview/<jobLink>`.
2. UI submits:
   - `POST /api/open-interviwe/create-interview`
   - Payload: `{ name, email, interviewId }`
3. Backend `createInterview`:
   - Validates job exists and active
   - Checks recruiter wallet minimum minutes threshold
   - Creates `InterviewSession`
   - Builds system prompt with job config and selected questions
   - Requests OpenAI Realtime session key
   - Returns `ephemeralKey`, `sessionId`, `duration`, `totalQuestions`

Route definition:

- `backend/routes/interviewSessions.js`
  Controller:
- `backend/controllers/interviewSessionController.js` (`createInterview`)

### 4.3 Live interview runtime

Frontend (`frontend/pages/interview/[interviewId].js`) performs:

- WebRTC SDP handshake directly to OpenAI Realtime endpoint
- Local anti-cheat/red-flag event detection
- Periodic red-flag upload:
  - `POST /api/mood-and-redflags`
- Local recording as 10s `webm` chunks, uploaded by:
  - `POST /api/upload-chunk`

Important behavior:

- AI response tags `[QUESTION]`, `[DISCUSSION]`, `[INTERVIEW_END]` are parsed client-side for flow control.
- Interview stops when tag indicates end or max/time logic triggers.

### 4.4 Interview finalization and analysis

On stop:

1. Frontend sends transcript batch:
   - `POST /api/transcription/bulk`
   - Includes `finalSubmission: true`
2. Backend `saveTranscript`:
   - Saves transcription into `InterviewSession.fullTranscription`
   - Calls OpenAI Chat Completions for structured analysis
   - Runs DB transaction:
     - Deduct recruiter minutes from wallet
     - Create `Transaction` usage ledger record
     - Mark session as `completed`, persist analysis
3. Background video chunk merge via ffmpeg (`processVideoAsync`).

Controller:

- `backend/controllers/interviewSessionController.js` (`saveTranscript`, `processVideoAsync`)

### 4.5 Recruiter review and exports

- Session listing: `GET /api/jobs/:id/sessions`
- Session analysis (current implementation): `GET /api/jobs/sessions/:sessionId/analysis`
- Results export CSV: `GET /api/jobs/:id/export`
- PDF export endpoint: `POST /api/export-analysis-pdf`

Frontend consumers:

- Job detail page: `frontend/pages/jobs/[id].js`
- Session list + analysis modal: `frontend/components/jobs/InterviewSessionList.js`, `frontend/components/analysis/AnalysisModal.js`

## 7. Operational Dependencies

- MongoDB running and reachable
- OpenAI API access enabled
- ffmpeg installed on backend host (for chunk merge)
- Puppeteer runtime dependencies (for PDF generation)
- SMTP credentials for invitation emails

## 8. Quick Runbook (Dev)

1. Start backend (`npm run dev` in `backend/`).
2. Start frontend (`npm run dev` in `frontend/`).
3. Create job and generate questions in UI.
4. Open candidate link `/interview/<interviewLink>`.
5. Complete interview and verify:
   - session status changes to completed
   - analysis exists
   - recruiter wallet minutes decrease
   - transaction ledger record exists
   - merged video file is created in uploads.
